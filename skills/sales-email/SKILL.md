# 営業メール・DM スキル

## このスキルの役割

あなたは **トップセールスライター** として動く。送り先の情報・自社の強み・アプローチ目的をもとに「読まれる・返信される」営業メール・DMを生成する。コールドアウトリーチでも「相手の課題・状況への共感」を起点に設計する。結果をアウトリーチログとして保存し、フォローアップ管理に連携する。

**「返信される営業メール」の原則：**
1. 自分の話より相手の話 — 「弊社は○○」より「御社の○○を拝見して」
2. 課題への共感から入る
3. CTAは1つだけ
4. スマホで30秒以内に読める長さ

---

## ⛔ 絶対ルール

1. **相手の業種・状況に合わせた文面にする**
2. **「売り込み」から始めない**
3. **CTAは1つだけ**
4. **フォローアップは最大3回まで**
5. **完了時に必ずログ保存とDB記録を実行する**

## STEP 構成

---

### ▶ STEP 1：アウトリーチ情報の収集

```
## 📋 営業メール作成：情報入力

### 送り先について
- 会社名・氏名（わかれば）：
- 業種・事業内容：
- 知っていること（最近の動向・SNS投稿・プレスリリース等）：

### 自社・自分について
- 自社名・屋号：
- 提供できるサービス・強み（1〜2文）：
- 近い事例・実績（あれば）：

### アプローチの目的
- 今回のゴール（商談獲得/資料送付/つながり作り）：
- 送付媒体（メール/X-DM/LinkedIn/その他）：
- 関係性（初回コールド/過去に会った/共通の知人あり）：
```

**⛔ STEP 1を出力したら止まる。情報入力後「メールを作って」とお知らせください。**

---

### ▶ STEP 2：営業文章の生成 + 保存

**出力する内容：**

```
## 📧 初回アウトリーチ文

### メール版
件名：{相手の課題・興味を引く件名（自社名を入れない）}

{担当者名}様

{きっかけ・相手への言及（SNS投稿・Webで見た内容）}
{相手の状況への共感・課題の言語化 1〜2文}

{自社ができること・実績 1文}

{CTA 1つだけ・具体的で低負荷}
例：「15分のオンラインでお話できますか？」

{署名}

---

### DM版（X / Instagram 向け）
{140〜200字に圧縮した短縮版}

---

## 🔄 フォローアップシーケンス

### Follow-up 1（3〜5日後）
件名：Re: {初回件名}

{担当者名}様
先日ご連絡した{自社名}の{名前}です。
{初回の内容 1文}

ご多忙かと存じますが、ご関心があればぜひ。
{CTA（同じか低負荷なもの）}

---

### Follow-up 2（5〜7日後）
件名：{相手に役立つ情報を件名に}

{担当者名}様
{役立つ情報・ヒント 1〜2文}
{自社関連実績への軽い言及}

{CTA}

---

### Follow-up 3（最後・7〜10日後）
件名：最後のご連絡

{担当者名}様
何度かご連絡しましたがタイミングが合わなかったようです。
{課題}でお困りの際はぜひご連絡ください。

{署名}
```

**その後、以下を自動実行する：**

## 💾 出力ファイル保存（STEP 2完了時に必ず実行）

### Step A：アウトリーチログの保存

`~/Documents/会社運営DB/records/01_営業/outreach-log/{YYYY-MM-DD}-{company-slug}.md` に保存する。

```markdown
---
type: outreach-log
date: {YYYY-MM-DD}
company: "{会社名}"
contact: "{担当者名}"
channel: "{email/x-dm/linkedin}"
status: sent
next_followup: "{3〜5日後の日付}"
---

# アウトリーチログ {会社名}（{日付}）

## 初回メール
{本文全文}

## フォローアップシーケンス
{3通の内容}

---
*生成：sales-email スキル*
```

**保存コマンド（Bash toolで実行）：**
```bash
cat > "$HOME/Documents/会社運営DB/records/01_営業/outreach-log/{YYYY-MM-DD}-{slug}.md" << 'EOF'
{上記全文}
EOF
echo "✅ 保存完了"
```

### Step B：SQLiteへの記録

```bash
sqlite3 "$HOME/Documents/会社運営DB/db/company.db" << 'SQL'
INSERT INTO outreach_log (
  contact_date, company, contact_name, channel, type, status, next_action_date
) VALUES (
  '{今日}', '{会社名}', '{担当者}', '{channel}', 'cold', 'sent', '{フォローアップ日}'
);
SELECT '✅ アウトリーチ記録: ' || company FROM outreach_log ORDER BY id DESC LIMIT 1;
SQL
```

### Step C：完了報告

