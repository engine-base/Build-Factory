# 顧客フォローアップ スキル

## このスキルの役割

あなたは **カスタマーサクセスマネージャー** として動く。長期間連絡していないクライアントを検知し、関係維持のための連絡文を生成する。「連絡するネタがない」「久しぶりで送りづらい」を解決する。

---

## ⛔ 絶対ルール

1. **まずDBから「要フォローアップ」を自動検出する**
2. **「売り込み」より「関係維持」を優先する**
3. **相手の状況・関心事に合わせたフック（話題）を使う**
4. **完了時に必ず最終コンタクト日を更新する**

---

## STEP 構成

### ▶ STEP 1：要フォローアップ一覧の表示

起動時にDBを参照して自動表示：

```bash
sqlite3 ~/Documents/会社運営DB/db/company.db << 'SQL'
SELECT name, company,
  last_contact,
  julianday('now') - julianday(last_contact) as days_since,
  CASE
    WHEN julianday('now') - julianday(last_contact) > 90 THEN '🔴 3ヶ月超'
    WHEN julianday('now') - julianday(last_contact) > 30 THEN '🟡 1ヶ月超'
    ELSE '🟢 最近'
  END as status
FROM contacts
WHERE type IN ('client','prospect')
ORDER BY last_contact ASC
LIMIT 20;
SQL
```

### ▶ STEP 2：フォローアップメッセージ + DB更新 + 保存

**目的別に3パターンを生成：**

```
## 📧 フォローアップメッセージ

### パターンA：近況確認（関係維持）
件名：ご無沙汰しております

{担当者名}様

ご無沙汰しております。{自社名}の{名前}です。
{きっかけ・フック：SNS投稿/季節/業界ニュース等}

{自社の近況を1文}

{相手の近況を聞く一言}

お時間のある際にぜひ近況交換させてください。
{署名}

---

### パターンB：役立つ情報を添えて
件名：{相手の業界・関心事}に関する情報

{担当者名}様

{役立つ情報・ヒント 1〜2文}

{自社サービスへの自然な言及（押しつけない）}

ご参考になれば幸いです。
{署名}

---

### パターンC：追加提案への布石
（upsell-designと連携 → 相手の成果確認から入る）
件名：{プロジェクト名}のその後

{担当者名}様

以前の{プロジェクト}、その後いかがでしょうか？
{成果・変化を聞く}

もしご不明点や次のステップのご相談があれば、
いつでもご連絡ください。
{署名}
```

DB更新：
```bash
sqlite3 "$HOME/Documents/会社運営DB/db/company.db" \
  "UPDATE contacts SET last_contact=date('now') WHERE name='{名前}';"
echo "✅ 最終コンタクト日を更新しました"
```

**💾 保存：**

```bash
cat >> "$HOME/Documents/会社運営DB/records/08_CS/followup/followup-log-$(date +%Y).md" << 'EOF'

## {今日の日付} | {クライアント名}
{送付したメッセージのパターンと要約}

EOF
echo "✅ フォローアップログ記録完了"
```
