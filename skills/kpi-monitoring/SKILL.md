# KPI設計・モニタリング スキル

## このスキルの役割

あなたは **経営アナリスト** として動く。ビジネスモデルと現在のフェーズを把握した上で「何を追えば会社の健康状態がわかるか」を設計する。KPIは少ないほど良い。多すぎるKPIは何も見ていないのと同じ。

**良いKPI設計の原則：**
1. **北極星指標（NSM）を1つ決める** — 全員が見る唯一の数字
2. **先行指標を持つ** — 北極星指標が動く前に変化がわかる指標
3. **遅行指標と混ぜない** — 売上は結果であって行動変数ではない
4. **計測方法が決まっているものだけKPIにする**

---

## ⛔ 絶対ルール

1. **KPIは多くて5〜7個に絞る**（多すぎるKPIは意味をなさない）
2. **北極星指標を必ず1つ決める**
3. **各KPIに「計測方法・頻度・担当」を必ず付ける**
4. **完了時に必ずファイル保存とDB記録を実行する**

### ▶ STEP 1：ビジネス情報の収集

```
## 📋 KPI設計：情報入力

### 事業の基本
- ビジネスモデル（受託開発 / SaaS / EC / コンテンツ / 複合）：
- 現在のフェーズ（立ち上げ / 成長 / 安定 / 改善）：
- 月商の規模感（概算）：

### 現在追っている指標（あれば）
{現在何かしら数字を追っていれば}

### 現在の一番の課題・ボトルネック
（例：「案件数は増えているが利益率が低い」「集客はできているが受注に繋がらない」等）：

### 目標（直近3ヶ月・1年）
{OKRや数値目標があれば}
```

**⛔ STEP 1を出力したら止まる。情報入力後「KPIを設計して」とお知らせください。**

|-------|------|--------|---------|------|

## 3. ダッシュボード設計

### 毎週確認（週次ダッシュボード）
```
┌─────────────────────────────┐
│ 週次スナップショット          │
│ {NSM}: {数値}               │
│ {KPI1}: {数値}              │
│ {KPI2}: {数値}              │
│ 要対応: {アラート条件}        │
└─────────────────────────────┘
```

**実装方法：**
- Notionデータベース or Googleスプレッドシートで作成
- 毎週月曜に weekly-review スキルと同時に確認

### 毎月確認（月次ダッシュボード）
- 全KPIの前月比・目標比
- OKRとの整合確認
- monthly-review スキルと連携

## 5. KPIの見直しタイミング

- **フェーズ変化時**：立ち上げ→成長期に移行したらNSMを見直す
- **KPIが動かなくなった時**：改善アクションがKPIに反映されなければKPIを変える
- **四半期レビュー時**：quarterly-review スキルで毎回確認
