# データ・セキュリティ管理

## このスキルがやること
事業データのバックアップ・アカウント管理・情報漏洩対策を設計・定期確認する。
クライアントの機密情報を扱う立場として、セキュリティは「オプション」ではなく「義務」。

---

## バックアップ設計（3-2-1ルール）

```
3: データのコピーを3つ持つ
2: 2種類の異なる媒体に保存（クラウド + ローカル）
1: 1つはオフサイト（別の場所）に保存

実装例：
  - ローカルPC（作業データ）
  - Google Drive / Dropbox（自動同期）
  - 外付けHDD or 別クラウドサービス
```

---

## アカウント管理

**パスワード管理：**
- パスワードマネージャーを1つ選んで全アカウントを集約（1Password / Bitwarden等）
- 重要サービスには必ず2段階認証を設定
- 「使い回しパスワード」を完全に廃止

**アカウント台帳の整理：**
- 全サービスのID・メールアドレス・2FA方法を1箇所に記録
- BCPと連携：緊急時に第三者が安全にアクセスできる手順

---

## 情報漏洩対策

**クライアント情報の取り扱い：**
- NDA締結後のみ機密情報を受領
- クライアントデータを個人のSNS・チャットに貼らない
- プロジェクト終了後のデータ削除手順を契約時に確認

**定期チェックリスト（月1回）：**
```
□ 不要なアカウントを解約・データを削除したか
□ 共有リンクが意図せず公開になっていないか
□ 使っていないアプリの権限を見直したか
□ OSとアプリを最新バージョンに更新したか
```
