# 顧客・売上依存リスク管理

## このスキルがやること
特定の顧客・業種・サービスへの売上依存度を数値で把握し、リスクを分散する計画を立てる。
「メイン顧客1社が契約を終了したら売上が半分になる」状態を把握・解消する。

---

## 危険な依存パターン

```
【顧客依存】
  → 1社で売上の50%以上を占めている状態
  → その会社の担当者が変わると契約終了リスク

【業種依存】
  → 顧客が特定業種（例：飲食業）に集中
  → その業種が不況になると全体が落ちる

【サービス依存】
  → 収益の全てが「受託開発」1本
  → 人を使った時間売りは上限がある

【チャネル依存】
  → 集客の全てが「紹介」1本
  → 紹介者との関係が変わると集客ゼロに
```

---

## 月次チェック項目

```
顧客別売上比率（03_財務・経理のP&Lから）
  顧客A: ○○円（全体の○○%）
  顧客B: ○○円（全体の○○%）
  その他: ○○円（全体の○○%）

警告ライン：1社30%超え → 分散計画を立てる
危険ライン：1社50%超え → 今月中に対策開始
```

---

## 分散計画の設計

**インプット：** 顧客別売上比率・業種・チャネル

**アウトプット：**
- 依存リスクの可視化レポート
- 新規顧客獲得の優先ターゲット定義（依存している業種・チャネルと異なる方向）
- 収益モデルの多角化案（受託 + 月額保守 + コンテンツ等）

---
