# 記事管理・コンテンツ監査

## このスキルがやること
公開済みの記事・コンテンツのパフォーマンスを追跡し、更新・削除・統合の優先順位を判定する。
write-article.skillで作った記事を「育てる・整理する・撤退する」管理の司令塔。

---

## 管理台帳の設計

各記事に以下を記録・追跡：

| 項目 | 内容 |
|------|------|
| URL | 記事のパス |
| メインキーワード | 狙っているKW |
| 公開日 / 最終更新日 | - |
| 検索順位 | 毎月更新 |
| 月間PV | GA4から取得 |
| 平均滞在時間 | - |
| CTR | Search Console |
| ステータス | 最新 / 要更新 / 要削除 / 統合候補 |

---

## 月次コンテンツ監査（月初 / 所要30〜60分）

**インプット**
- GA4・Search Console データ
- 記事管理台帳（前月版）

**アウトプット**
- 各記事のステータス判定（下記基準）
- 今月の更新優先リスト（Top 5）
- 削除・統合候補リスト
- 新規記事が必要なキーワードリスト → write-article.skillへ

### 判定基準

```
【最新】     流入あり・順位安定 → 維持
【要更新】   順位下落 / 情報が古い / PV落ち → 今月中に更新
【統合候補】 似たテーマの記事が複数 → 1本にまとめてカニバリ解消
【要削除】   PV月10以下・順位圏外・古すぎ → noindex or 削除
```

---

## write-article.skillからの引き継ぎフォーマット

write-article.skillで記事を生成したら、以下を管理台帳に登録：
- URL・タイトル・メインKW・公開日
- ターゲット順位（目標）
- 初回確認予定日（公開から1ヶ月後）
