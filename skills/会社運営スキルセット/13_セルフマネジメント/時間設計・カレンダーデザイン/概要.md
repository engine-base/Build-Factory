# 時間設計・カレンダーデザイン

## このスキルがやること
1週間のカレンダーを設計し、「集中できる時間帯」に「最重要な仕事」を割り当てる。
「気づいたら1日がミーティングと返信で終わった」を構造的に防ぐ。

---

## 時間の3分類

```
【創造的時間】Deep Work（最も集中力が必要な仕事）
  例：開発・提案書作成・コンテンツ制作・戦略思考
  → 1日のピーク時間帯（人によって午前中 or 夕方）に2〜4時間確保
  → 通知オフ・単一タスク

【管理的時間】Shallow Work（処理・確認・返信）
  例：メールチェック・請求書確認・SNS投稿
  → 1日2回に集約（朝9時・夕17時等）
  → 創造的時間を邪魔させない

【バッファ時間】予期しない対応・休息・振り返り
  → 1日の15〜20%を意図的に空けておく
  → 埋めない。緊急対応・思考整理に使う
```

---

## 週次カレンダーテンプレート設計

**インプット：** 自分の集中できる時間帯・固定の約束ごと

**アウトプット：**
```
【月曜】週次スタート：週次計画・最重要タスクDay1
【火〜水】Deep Work集中日（商談は極力入れない）
【木曜】外部ミーティング日（商談・MTGをここに集約）
【金曜】週次レビュー・管理業務・来週準備
```

**固定ブロックの設計例：**
```
毎朝 8:30〜9:00  朝のルーティン（Top3決め）
毎朝 9:00〜12:00 Deep Work（最重要タスク）
毎日 12:00〜13:00 昼休憩
毎日 13:00〜17:00 中作業・ミーティング・管理業務
毎日 17:00〜17:30 メールチェック・返信
毎週金 16:00〜17:00 週次レビュー
```

---

## 「ミーティング日」の集約

商談・打ち合わせを特定の曜日に集める設計で、他の日の集中を守る。
クライアントへの提案方法：「打ち合わせは木曜がご都合合いますでしょうか？」
