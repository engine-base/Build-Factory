# 意思決定・優先度フレームワーク

## このスキルがやること
「やる / やらない」「今やる / 後でやる」「自分でやる / 外注する」の判断基準を明文化する。
一人会社は意思決定の全てが自分にかかる。基準を持たないと判断疲れで質が下がる。

---

## 主要な判断フレームワーク

### 1. 新規依頼・案件の受注判断
```
以下を全て確認してから決める：
□ 年次OKR・四半期OKRと合致しているか
□ 利益率は最低ラインを超えているか
□ 今のキャパシティで受けられるか
□ 断った場合の機会損失 vs 受けた場合の疲弊
□ 3年後の自分が「あの仕事を受けてよかった」と思えるか

→ 2つ以上Noなら断る。迷ったら断る。
```

### 2. 新しいツール・サービスの導入判断
```
□ 今の課題を具体的に解決するか（「便利そう」だけでは×）
□ 月額コストは投資対効果に見合うか
□ 既存ツールで代替できないか
□ 無料トライアルを使ったか

→ 3ヶ月試してROIが出なければ解約
```

### 3. 自分でやる vs 外注の判断
```
時給換算して判断する：
自分の時給 = 月収 ÷ 労働時間

タスクの時間 × 自分の時給 > 外注費用
  → 外注が合理的

AND 以下の条件も確認：
□ 自分がやることで品質が大きく上がるか
□ そのタスクを自分がやることで何かを学べるか
□ 外注できる信頼できる人がいるか
```

### 4. 今やる vs 後でやるの判断（2分ルール）
```
2分以内に終わる → 今すぐやる
2分以上かかる → タスクリストに入れて後でまとめてやる
```
