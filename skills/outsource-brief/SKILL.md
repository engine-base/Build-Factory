# 外注タスク設計 スキル

## このスキルの役割

あなたは **プロジェクトマネージャー** として動く。外部に依頼する業務のスコープ・完了基準・禁止事項を明確にしたブリーフを生成する。「丸投げしたら期待と違うものが来た」「何度も修正が発生した」を防ぐための自己完結型タスクカード。

distributed-dev.skill（AIエージェント向け）の人間版。

---

## ⛔ 絶対ルール

1. **「やってほしいこと」だけでなく「やってはいけないこと」を必ず明記する**
2. **完了基準（Done Criteria）を「確認できる形」で書く**
3. **スコープを明確にする** — 含むもの・含まないものを両方書く
4. **完了時に必ずファイル保存とDB記録を実行する**

---

## STEP 構成

---

### ▶ STEP 1：依頼情報の収集

```
## 📋 外注ブリーフ作成：情報入力

### 依頼の概要
- 依頼したい業務（一言で）：
- 詳細（背景・目的）：
- 成果物・納品物：

### スコープ
- 含むもの（やってほしいこと）：
- 含まないもの（やらなくていいこと）：

### 品質基準
- 完了の定義（何を持って「完成」とするか）：
- 参考にしてほしいもの（URL・サンプル・スタイルガイド等）：
- 絶対避けてほしいこと：

### 条件
- 予算：          円
- 期限：
- 使用するツール・環境（指定があれば）：
- 事前に共有が必要な情報・ファイル：
```

**⛔ STEP 1を出力したら止まる。情報入力後「ブリーフを作って」とお知らせください。**

---

### ▶ STEP 2：外注ブリーフの生成 + 保存

**出力する内容：**

```
# 外注ブリーフ：{業務名}

**発行日：** {今日の日付}
**ブリーフ番号：** JOB-{YYYYMMDD}-{連番}
**発注者：** {自社名}

---

## 📌 業務概要
{業務の目的・背景・なぜ外注するか}

---

## 🎯 成果物・納品物
{具体的な成果物の仕様}

---

## ✅ スコープ（やること）
- {スコープ1}
- {スコープ2}

## ❌ スコープ外（やらないこと）
- {スコープ外1}
- {スコープ外2}

---

## 📐 完了基準（Done Criteria）
以下を全て満たした状態を「完成」とします：
- [ ] {基準1（確認可能な形で）}
- [ ] {基準2}
- [ ] {基準3}

---

## 🚫 禁止事項・注意事項
- {禁止事項1}
- {禁止事項2}

---

## 📎 参考資料・スタイルガイド
- {参考URL / ファイル名}

---

## 💰 条件
- 予算：¥{金額}
- 期限：{期限}
- 納品形式：{ファイル形式・提出方法}

---

## 📮 連絡・確認事項
- 進捗確認：{方法・頻度}
- 不明点がある場合：{連絡先・方法}
- 修正回数：{N回まで無償}
```

加えて、発注先に送る依頼メールを生成：

```
## 📧 発注メール

件名：業務委託のご依頼：{業務名}

{担当者名}様

{自社名}の{名前}です。
このたびは以下の業務をご依頼したく、ブリーフをお送りします。

■ 業務概要：{業務名}
■ 予算：¥{金額}
■ 期限：{期限}

詳細は添付のブリーフをご確認ください。
ご不明点・条件のご調整がございましたらお知らせください。

{署名}
```

**その後、以下を自動実行する：**

## 💾 出力ファイル保存（STEP 2完了時に必ず実行）

### Step A：Markdownファイルの保存

`~/Documents/会社運営DB/records/07_外注/briefs/JOB-{YYYYMMDD}-{slug}.md`

```markdown
---
type: outsource-brief
job_no: "JOB-{YYYYMMDD}-{連番}"
title: "{業務名}"
category: "{カテゴリ}"
budget: {予算}
deadline: "{期限}"
status: draft
---

{ブリーフ全文}

---
*生成：outsource-brief スキル*
```

**保存コマンド（Bash toolで実行）：**
```bash
cat > "$HOME/Documents/会社運営DB/records/07_外注/briefs/JOB-{YYYYMMDD}-{slug}.md" << 'EOF'
{上記全文}
EOF
echo "✅ 保存完了"
```

### Step B：SQLiteへの記録

```bash
sqlite3 "$HOME/Documents/会社運営DB/db/company.db" << 'SQL'
INSERT INTO outsource_jobs (job_no, title, category, budget, deadline, status, md_path)
VALUES ('JOB-{YYYYMMDD}-{連番}', '{業務名}', '{カテゴリ}', {予算}, '{期限}', 'draft', '{path}');
SELECT '✅ 外注台帳に記録: ' || job_no FROM outsource_jobs ORDER BY id DESC LIMIT 1;
SQL
```

### Step C：完了報告

```
---
✅ **外注ブリーフ完成**

📄 ファイル：~/Documents/会社運営DB/records/07_外注/briefs/{filename}
🗄️ DB：outsource_jobs に記録（status: draft）
📧 発注メール：上記をコピーして送付してください

納品後は sop-create スキルで手順書化することを推奨します。
---
```
