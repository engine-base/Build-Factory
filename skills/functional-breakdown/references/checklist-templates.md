# チェックリストテンプレート

各種別の項目を `decided` に昇格させるために潰すべきチェックリスト。STEP 2 で各項目を開いた際、AI はこのテンプレートに従って初期チェックリストを生成する。プロジェクト固有の項目は追加して良いが、ここに列挙された項目は **削除しない** こと。

---

## 画面項目 (S-XXX) の標準チェックリスト

| Key | 確認事項 | 例 / 提案ガイド |
|---|---|---|
| CL-S-1 | 表示項目 (どのフィールドを出すか) | 商品名・価格・在庫・画像・レビュー件数 |
| CL-S-2 | レイアウト | grid / table / card / form / hybrid のいずれか |
| CL-S-3 | 操作ボタン (露出する CRUD) | 検索・並び替え・お気に入り・カート追加 |
| CL-S-4 | 検索/フィルタ/ソートの対象フィールド | カテゴリ・価格帯・在庫あり・新着順 |
| CL-S-5 | ページング方式 | 通常ページネーション / 無限スクロール / 件数指定 |
| CL-S-6 | 状態の表示 | loading / empty / error / unauthorized |
| CL-S-7 | 遷移元・遷移先 | ホーム → S-001 → S-002 (商品詳細) |
| CL-S-8 | レスポンシブ対応 | PC / SP / タブレットでの出し分け |
| CL-S-9 | 閲覧権限 (どのロールが見える) | R-001 (ゲスト) / R-002 (一般) |
| CL-S-10 | 編集/削除権限 | R-003 (管理者) のみ |
| CL-S-11 | 関連 API エンドポイント候補 | `GET /api/products` |
| CL-S-12 | 関連エンティティ | E-002 (Product) |

## 認証機能 (F-XXX かつ category=auth) の追加チェックリスト

機能項目の標準 10 個に **追加で** 8 項目を必ず含める。

| Key | 確認事項 | 例 / 提案ガイド |
|---|---|---|
| CL-AUTH-1 | ログイン手段 | email + password / Google / GitHub / Magic Link / SAML SSO |
| CL-AUTH-2 | パスワード強度・有効期限 | 12 文字 + 大小英数記号 / 期限 90 日 / 履歴 5 個禁止 |
| CL-AUTH-3 | パスワードリセット手段 | メールでトークン (有効 30 分) / SMS / セキュリティ質問 |
| CL-AUTH-4 | 2FA | TOTP / SMS / バックアップコード / 任意 vs 必須 / 管理者は必須 |
| CL-AUTH-5 | セッション管理 | TTL 24h / リフレッシュトークン 7d / 同時ログイン上限 N |
| CL-AUTH-6 | アカウントロック | 試行 5 回で 15 分ロック / 解除はメール認証 |
| CL-AUTH-7 | 退会フロー | 即時削除 / 30 日猶予 / データ保持期間 / 法令保管項目除外 |
| CL-AUTH-8 | ソーシャルログイン紐付け解除 | パスワード設定済みの場合のみ可 / 最後の 1 つは解除不可 |

## チェックリスト生成のガイド

AI が初期ドラフト時に自動生成する際は:

1. **要件定義の文脈から具体例を入れる** — テンプレ通りではなく「このプロジェクトでは商品名・在庫数・サムネ画像」など
2. **業界規制を反映** — EC なら特商法表示項目を CL-S-1 に必ず含める / 医療なら CL-AUTH-4 に 2FA 必須を提案
3. **AI 推奨案を必ず付ける** — チェックリスト各項目に「→ 提案: ...」を添えてユーザーが OK と言いやすい状態にする
4. **重要度マーク** — 必須 (★) / 推奨 (◆) / 任意 (○) を頭に付ける
