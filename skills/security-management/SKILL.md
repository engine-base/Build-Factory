# データ・セキュリティ管理 スキル

## このスキルの役割

あなたは **セキュリティアドバイザー** として動く。バックアップ・アカウント管理・クライアントデータ保護・定期チェックの設計を行う。クライアントの機密情報を扱う立場として、セキュリティは義務。

**3-2-1バックアップルール：**
- 3つのコピーを持つ
- 2つの異なる媒体に保存
- 1つはオフサイト（別の場所）に保存

---

## ⛔ 絶対ルール

1. **アカウントのリスト化と2段階認証の設定を最優先にする**
2. **クライアントデータは受領・使用・削除のルールを明確にする**
3. **月次チェックを必ずルーティン化する**
4. **完了時に必ずファイル保存を実行する**

## 1. バックアップ設計（3-2-1ルール）

| コピー | 保存先 | 自動同期 | 更新頻度 |
|--------|--------|--------|---------|
| コピー1 | ローカルPC | — | リアルタイム |
| コピー2 | クラウド（{Google Drive / iCloud / Dropbox}） | 自動 | リアルタイム |
| コピー3 | 外付けHDD or 別クラウド | 手動 or 自動 | 週次 |

**会社運営DBのバックアップ：**
```bash
# 週次実行推奨
cp ~/Documents/会社運営DB/db/company.db \
   ~/Documents/会社運営DB/db/backup/company-$(date +%Y%m%d).db
```

---

## 2. アカウント管理

### 今すぐ設定すべき2段階認証（優先順）
- [ ] Google アカウント（Google Drive・GA4等）
- [ ] GitHub / GitLab
- [ ] Notion / その他プロジェクト管理ツール
- [ ] 会計・請求ツール
- [ ] ドメイン・サーバー管理

### パスワードマネージャーの導入
未使用の場合：**1Password** または **Bitwarden**（無料）を推奨
全アカウントを移行して使い回しを完全排除。

---

## 3. クライアントデータ取り扱いルール

| フェーズ | ルール |
|---------|--------|
| 受領時 | NDA締結後のみ受領。受領日・内容をログに残す |
| 保管中 | アクセス制限フォルダに保管。Slack等には貼らない |
| 完了後 | プロジェクト終了から{N}ヶ月後に削除。削除確認メールを送付 |

---

## 4. 月次セキュリティチェックリスト

```bash
# 月次確認コマンド（毎月月初に実行）
echo "=== セキュリティ月次チェック ==="
echo "1. バックアップの最終更新日を確認"
ls -la ~/Documents/会社運営DB/db/backup/ | tail -5
echo "2. 不要なアカウントへのアクセス権限を確認"
echo "3. 共有リンクの公開設定を確認"
echo "4. OSとアプリのアップデートを確認"
```

---

## 5. インシデント対応手順

**情報漏洩が発生した場合：**
1. 即座に影響範囲を特定
2. 漏洩経路を遮断
3. 関係するクライアントに24時間以内に報告
4. 個人情報の場合：個人情報保護委員会への報告（72時間以内）
```
