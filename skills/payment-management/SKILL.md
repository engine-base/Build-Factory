# 入金管理・催促 スキル

## このスキルの役割

あなたは **経理担当者** として動く。SQLiteの請求台帳を参照して未払い一覧を表示し、入金確認・催促メール生成・ステータス更新を行う。「請求したのに入金されていない」を見逃さない仕組み。

---

## ⛔ 絶対ルール

1. **まず未払い一覧をDBから取得して表示する**
2. **期限超過（overdue）と未払い（unpaid）を区別する**
3. **催促メールは段階的に強度を上げる（1回目は柔らかく）**
4. **入金確認後は必ずDBのstatusを更新する**

---

## STEP 構成

---

### ▶ STEP 1：未払い一覧の取得と確認

起動時にまずDBを確認して現状を表示する。

**Bash toolで以下を実行してから出力する：**

```bash
sqlite3 ~/Documents/会社運営DB/db/company.db << 'SQL'
SELECT
  invoice_no,
  client,
  total,
  issued_date,
  due_date,
  status,
  CASE
    WHEN due_date < date('now') AND status='unpaid' THEN '🚨 期限超過'
    WHEN due_date BETWEEN date('now') AND date('now', '+7 days') AND status='unpaid' THEN '⚠️ 期限間近'
    ELSE '📋 未払い'
  END as alert
FROM invoices
WHERE status IN ('unpaid','overdue')
ORDER BY due_date;
SQL
```

**取得結果をもとに以下を出力する：**

```
## 💰 入金管理ダッシュボード（{今日の日付}時点）

### 未払い請求書一覧

| 請求書番号 | クライアント | 金額 | 支払期限 | 状況 |
|----------|-----------|------|---------|------|
| {INV番号} | {会社名} | ¥{金額} | {期限} | {🚨期限超過/⚠️期限間近/📋未払い} |

### サマリー
- 未払い合計：¥{合計}（{件数}件）
- 期限超過：{件数}件
- 今週期限：{件数}件

---
何をしますか？
A. 入金確認（支払われた請求書のステータスを更新）
B. 催促メールを作成
C. 特定の請求書の詳細を確認
```

**⛔ STEP 1を出力したら止まる。A/B/Cを選択してください。**

---

### ▶ STEP 2A：入金確認とステータス更新

```
## ✅ 入金確認

入金が確認できた請求書番号を教えてください：
（複数ある場合はすべて）

入金日（通帳で確認できた日）：
```

情報を受け取ったら以下を実行：

```bash
sqlite3 ~/Documents/会社運営DB/db/company.db << 'SQL'
UPDATE invoices
SET status='paid', paid_date='{入金日}'
WHERE invoice_no='{INV番号}';
SELECT '✅ 入金確認済み: ' || invoice_no || ' / ' || client || ' ¥' || total
FROM invoices WHERE invoice_no='{INV番号}';
SQL
```

完了報告：
```
---
✅ **入金確認完了**
🗄️ {INV番号}（{クライアント}）¥{金額} → status: paid に更新
📅 入金日：{入金日}
---
```

---

### ▶ STEP 2B：催促メールの生成

```
## 📧 催促メールの生成

催促する請求書番号：
催促の回数（初回 / 2回目 / 最終）：
```

情報を受け取ったら以下を出力：

**1回目催促（柔らかい確認）：**
```
件名：【ご確認】{INV番号} お支払いのご確認

{担当者名}様

いつもお世話になっております。{自社名}の{名前}です。

{発行日}にお送りした下記請求書の入金を確認できておりません。
お手元に届いていない場合はご連絡ください。

■ 請求書番号：{INV番号}
■ 請求金額：¥{金額}（税込）
■ 支払期限：{期限}

お振込先：{銀行情報}

ご多忙のところ恐れ入りますが、ご確認いただけますと幸いです。
{署名}
```

**2回目催促（明確な催促）：**
```
件名：【再ご連絡】{INV番号} お支払いのお願い

{担当者名}様

以前ご連絡した請求書につき、まだ入金を確認できておりません。
お手数ですが今週中にお振込みいただけますでしょうか。

■ 請求書番号：{INV番号}
■ 請求金額：¥{金額}（税込）
■ 支払期限：{期限}（{超過日数}日超過）

{銀行情報}

ご不明点がございましたらお知らせください。
{署名}
```

**最終催促：**
```
件名：【重要】{INV番号} お支払いの最終ご確認

{担当者名}様

度重なるご連絡となり恐縮ですが、以下請求書の入金確認ができておりません。
{今日+7日}までにお振込みがない場合、別途対応を検討せざるを得ません。

■ 請求書番号：{INV番号}
■ 請求金額：¥{金額}
■ 当初支払期限：{期限}

お急ぎでご対応いただけますよう、お願い申し上げます。
{署名}
```

催促メール送付後、DBのstatusをoverdue更新：
```bash
sqlite3 ~/Documents/会社運営DB/db/company.db \
  "UPDATE invoices SET status='overdue' WHERE invoice_no='{INV番号}';"
```

## 月次入金サマリー

```bash
sqlite3 ~/Documents/会社運営DB/db/company.db \
  "SELECT strftime('%Y-%m',paid_date) as month, COUNT(*) as count, SUM(total) as total
   FROM invoices WHERE status='paid' GROUP BY month ORDER BY month DESC LIMIT 6;"
```
