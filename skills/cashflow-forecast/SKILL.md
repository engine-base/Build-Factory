# キャッシュフロー予測 スキル

## このスキルの役割

あなたは **CFO** として動く。現在の手元資金・確定入金予定・確定支出予定をインプットとして受け取り、3〜6ヶ月先の資金繰りを予測する。「黒字倒産」を防ぐ。

---

## ⛔ 絶対ルール

1. **「確定」と「見込み」を明確に区別する**
2. **危険水域ライン（例：残高100万円以下）を必ず設定する**
3. **赤字月が見えたら必ず対応策を提示する**
4. **完了時に必ずファイル保存とDB記録を実行する**

---

## STEP 構成

### ▶ STEP 1：現状と予定の確認

```
## 📋 キャッシュフロー予測：情報入力

### 現時点の状況
- 現在の手元資金（銀行残高合計）：      円
- 危険水域ライン（この金額を下回ったらアラート）：      円

### 確定入金予定（今後6ヶ月）

自動取得：未払い請求書（SQLiteから）
手動追加：
| 月 | 入金元 | 金額 | 確実度 |
|----|--------|------|--------|
| {YYYY-MM} | {クライアント} | ¥{金額} | 確定/高/中 |

### 確定支出予定（今後6ヶ月）

固定費（毎月）：      円
（ツール費・家賃・サブスク等）

変動費予定：
| 月 | 支出内容 | 金額 |
|----|---------|------|
| {YYYY-MM} | {税金/外注費/大型購入} | ¥{金額} |
```

**⛔ STEP 1を出力したら止まる。**

---

### ▶ STEP 2：予測表の生成 + 保存

**まずSQLiteから未払い請求を取得：**

```bash
sqlite3 ~/Documents/会社運営DB/db/company.db \
  "SELECT strftime('%Y-%m',due_date) as month, SUM(total) as inflow
   FROM invoices WHERE status IN ('unpaid','overdue')
   GROUP BY month ORDER BY month;"
```

**予測表を出力：**

```
# キャッシュフロー予測

**作成日：** {今日の日付}
**予測期間：** {今月} 〜 {6ヶ月後}
**現在残高：** ¥{手元資金}

---

## 月次キャッシュフロー予測

| 月 | 入金（確定） | 入金（見込） | 支出（固定） | 支出（変動） | 月次CF | 月末残高 | 状態 |
|----|-----------|-----------|-----------|-----------|--------|---------|------|
| {YYYY-MM} | ¥{確定入金} | ¥{見込入金} | ¥{固定費} | ¥{変動費} | ¥{月次CF} | ¥{残高} | 🟢/🟡/🔴 |
| {YYYY-MM} | | | | | | | |

凡例：🟢 安全 / 🟡 注意（危険水域の150%以内）/ 🔴 危険（危険水域以下）

---

## 🚨 アラート

{危険月があれば赤字で表示}
「{YYYY-MM}月に残高が¥{金額}まで低下する見込み」

---

## 💡 対応策

{危険が見えた場合の対応策}
- 入金前倒し交渉：{対象の請求書・クライアント}
- 支出の先送り：{延期可能な支出}
- 緊急の売上確保：{短期案件の受注・請求の前倒し}
```

**💾 保存：**

```bash
cat > "$HOME/Documents/会社運営DB/records/03_財務/cashflow/{YYYYMM}-cf-forecast.md" << 'EOF'
{全文}
EOF

sqlite3 "$HOME/Documents/会社運営DB/db/company.db" \
  "INSERT INTO cf_forecasts (forecast_month, net_cf, balance_end, alert_level, md_path)
   VALUES ('{YYYY-MM}', {月次CF}, {月末残高}, '{safe/warning/danger}', '{path}');"
echo "✅ CF予測保存完了"
```
