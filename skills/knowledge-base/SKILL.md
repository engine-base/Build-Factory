# ナレッジベース構築 スキル

## このスキルの役割

あなたは **ナレッジマネージャー** として動く。分散している情報を整理し、「あの情報どこだっけ」をなくす仕組みを設計する。会社運営DBのrecordsフォルダ自体がナレッジベースになっているが、このスキルはその設計・整理・検索を強化する。

---

## ⛔ 絶対ルール

1. **情報の「在り処」を一箇所に集中させる**（分散させない）
2. **保存ルール・命名規則を決めてから運用する**
3. **定期メンテナンス（月1）を習慣化する**
4. **完了時に必ずファイル保存とDB記録を実行する**

---

## STEP 構成

### ▶ STEP 1：モードの確認

```
**A. ナレッジベースの初期設計をする**

（Cの場合：検索したいキーワード・テーマを教えてください）
```

## 4. 検索の使い方

Obsidianでの検索：Cmd+Shift+F → キーワード検索
会社運営DBのSQLite検索：
```bash
# タイトルで検索
sqlite3 ~/Documents/会社運営DB/db/company.db \
  "SELECT title, category, md_path FROM knowledge_base WHERE title LIKE '%{キーワード}%';"

# 全文検索（recordsフォルダ）
grep -r "{キーワード}" ~/Documents/会社運営DB/records/ --include="*.md" -l
```
```

**💾 保存：**

```bash
cat > "$HOME/Documents/会社運営DB/records/09_情報/knowledge-base/KB-DESIGN-{YYYYMMDD}.md" << 'EOF'
{全文}
EOF

sqlite3 "$HOME/Documents/会社運営DB/db/company.db" \
  "INSERT INTO knowledge_base (title, category, summary, md_path)
   VALUES ('ナレッジベース設計書', 'design', '会社運営DBの情報整理設計', '{path}');"
echo "✅ ナレッジベース設計保存完了"
```
