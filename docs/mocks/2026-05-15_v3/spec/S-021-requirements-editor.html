2. 機能要件 (Must)

2.1 認証 (S-001〜005)

AC-001 (EARS: EVENT-DRIVEN)

When 有効な email + password が POST /api/auth/login に送られた時、 the system shall 200 と JWT + refresh_token cookie を返却する。

S-001·T-V3-AUTH-01
AC-002 (EARS: UNWANTED)

If credentials が不正な場合、 the system shall 401 と {"error":"invalid_credentials"} を返却する (user enumeration 防止)。

AC-003 (EARS: STATE-DRIVEN)

While user.mfa_enabled = TRUE の状態で、 the system shall 202 + mfa_challenge_id を返して /api/auth/mfa/verify を要求する。

※ AC は EARS notation (Easy Approach to Requirements Syntax) で書く。5 形式: UBIQUITOUS / EVENT-DRIVEN / STATE-DRIVEN / OPTIONAL / UNWANTED。

2.2 案件管理

アカウント配下に複数の workspace (案件) を作成・並列管理できる。各 workspace は独立した spec / task / session を持つ。

EARS Helper
UBIQUITOUS (常時)
The system shall ...
EVENT-DRIVEN
When [event], the system shall ...
STATE-DRIVEN
While [state], the system shall ...
OPTIONAL
Where [feature], the system shall ...
UNWANTED
If [condition], the system shall not ...
バリデーション
EARS 形式 OK (24/24)
