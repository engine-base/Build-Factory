# Build-Factory Constitution v4.2

## 1. 開発の根本原則

- **Test pass = done ではない**: AC を spec の逐語に対して満たして初めて done
- **3-tier AC を全 task で必須**: structural / functional / regression
- **mock 一致は機械検証**: scripts/lint-mock-impl-diff.sh で gate

## 2. 命名規約

- spec entity (PascalCase) ↔ DB table (snake_case) 1:1
- `bf_` prefix 禁止 (ADR-014)
- entities.json を single source of truth

## 3. セキュリティ (赤線)

- 本番 DB に `DROP` / `TRUNCATE` / `DELETE *` = 即セッション kill
- `.env` / 鍵ファイル commit = git pre-commit + secret scanning
- AGPL ライセンス依存追加 = 自動却下

## 4. AI 社員行動規範

- 完了報告は手動 audit MD を必ず添える (auto-generated 禁止)
- 仕様変更 / DB schema 変更 = masato 確認必須
- 軽微変更 (UI / 文言 / バグ修正) は自走 OK / git commit で報告

## 5. 意思決定の優先順位

1. ユーザー (松本) の判断基準を最優先
2. 仕様徹底 (mock = 実装 = spec)
3. テスト網羅
4. 速度 (※ 1〜3 を犠牲にしない)
