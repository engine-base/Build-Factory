受託EC構築 #4 — 要件定義書

本仕様書は、株式会社XX (発注者) と株式会社ENGINE BASE (受注者) における、独自 EC サイト構築プロジェクト 受託EC構築 #4 の機能・非機能要件を定める。

1. 機能要件

当 EC は、Shopify 等の汎用テンプレートでは対応できない、複雑な商品構成と取引形態を扱う。主要な機能要件は以下の通り。

1.1 商品管理

  • SKU 単位の在庫管理（複数倉庫対応）
  • 商品バリアント（色 × サイズ × 素材 の最大 3 軸）
  • 取引先別個別単価（卸価格 5 段階 + 取引先個別）

1.2 バンドル販売

複数 SKU を組み合わせた「バンドル商品」を販売できる。バンドル価格は、構成 SKU の合計単価から自動計算するか、固定価格を設定できる。

// バンドル定義の例
{
  "bundle_id": "BDL-001",
  "items": [
    { "sku": "SKU-A", "qty": 2 },
    { "sku": "SKU-B", "qty": 1 }
  ],
  "pricing": "fixed",   // fixed | sum
  "price": 12800
}

1.3 定期購入

  • 初回 / 2 回目以降 の単価を別設定
  • 配送間隔: 週次 / 隔週 / 月次 / カスタム
  • スキップ・解約は会員ページから自己操作可能

2. 非機能要件

  • P95 レイテンシ < 1.5s（同時接続 200 想定）
  • 稼働率 99.9% （月間ダウンタイム 43 分以下）
  • 個人情報は AES-256 暗号化保存（pgsodium）

5. 受け入れ条件 (EARS)

  • UBIQUITOUS: The system shall support BtoB pricing tiers for every product SKU.
  • EVENT: When a customer places a bundle order, the system shall deduct stock for each constituent SKU.
  • STATE: While a subscription is paused, the system shall not generate billing.
  • UNWANTED: If payment authorization fails, the system shall not ship the order.
