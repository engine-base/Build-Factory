# T-019-02: modify 対象 GitHub Issue 化

`scripts/scan-modify-targets.py` で検出した modify 候補を GitHub Issue として
登録するための運用文書。

## 概要

T-019-01 (bootstrap archive: onlook/penpot 削除) の続編。**将来 archive /
refactor / deprecate される候補** を機械的に検出 → 構造化 JSON 出力 →
GitHub Issue 化することで、削除候補をトラッカブルにする。

## 検出カテゴリ

| カテゴリ | 検出内容 |
|---|---|
| `deprecated_deps` | `requirements.txt` / `package.json` に DEPRECATED/REMOVE タグ |
| `stale_routers` | `backend/routers/*.py` で `backend/main.py` から `include_router` されていない module |
| `stale_services` | `backend/services/*.py` で backend 内から import されていない module |
| `archived_keyword` | ソース内に `# ARCHIVE` / `# DEPRECATED` / `TODO: remove` 等のタグがある行 |

## Usage

### 1. Scan (read-only, no side effects)

```bash
# 全カテゴリ → stdout に JSON
python3 scripts/scan-modify-targets.py

# 単一カテゴリで filter
python3 scripts/scan-modify-targets.py --category stale_routers

# ファイル出力
python3 scripts/scan-modify-targets.py --out docs/modify-targets/2026-05-12.json
```

### 2. GitHub Issue 化 (手動オペレーション)

scanner は **自動で Issue を作らない** (side effect 防止)。手動で `gh` CLI 経由で Issue 化:

```bash
# JSON ファイルから個別 Issue 作成
SCAN_JSON=docs/modify-targets/2026-05-12.json

# python3 で各 target を Issue に変換
python3 -c "
import json
data = json.load(open('${SCAN_JSON}'))
for t in data['targets']:
    title = f\"[modify-target] {t['category']}: {t['file']}\"
    body = f\"**Category**: {t['category']}\n**File**: {t['file']}:{t['line']}\n**Reason**: {t['reason']}\n**Snippet**:\n\`\`\`\n{t['snippet']}\n\`\`\`\n\nGenerated by T-019-02 scanner.\"
    print(f'gh issue create --title \"{title}\" --body \"{body}\" --label modify-target,refactor')
" | bash -x
```

### 3. ラベル運用

- `modify-target`: 全 Issue 共通
- `category:stale_router` / `category:stale_service` / `category:deprecated_dep` / `category:archived_keyword`: カテゴリ別
- 担当者は適宜 assign / priority label を追加

## 注意事項

- **stale_services** は false positive (test 経由でのみ使用 / dynamic import) を含む可能性あり → Issue 化前に手動 review 推奨。
- scanner 結果は **時点情報** (commit ごとに変動)。CI で定期実行 → 差分通知の運用も可能。
- 既存 lint-mock.sh / pre-commit-check.sh / license-check.yml / ci.yml は無改変 (REUSE)。本 scanner は独立 utility。
