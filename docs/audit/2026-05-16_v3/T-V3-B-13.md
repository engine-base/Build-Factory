## ノート
- **F-008 spec drift handling**: spec は workspace-scoped (/api/workspaces/{id}/phases) だが
  impl 既存 bf_phases table は project_id FK のみ持つ (E-012 spec_table_name='phases' /
  impl table_name='bf_phases' / impl FK=project_id)。本 ticket では既存 /api/projects/...
  API を破壊せず、workspaces.py に新規 endpoint を追加し、内部で workspace_id →
  primary bf_project.id を auto-resolve (services/phase_service.py::resolve_project_id_for_workspace)。
- **F-008 policies max_phases_per_workspace=10**: services/phase_service.py::
  MAX_PHASES_PER_WORKSPACE constant + count_phases_for_workspace で skipped 除外 count。
- **gate_conditions の永続化**: spec POST body の gate_conditions:string[] は
  bf_phases.notes に JSON `{"gate_conditions": [...]}` で永続化 (新 column 追加せず
  既存 schema 互換)。
- **gate 評価 MVP**: 現状の MVP では phase.status='completed' を gate pass の必要条件と
  し、未完了の場合は failing_conditions=['phase_not_completed' + each gate condition]
  で 409 を返す。`force=true` で workspace_admin が手動 unlock 可能。本格的な task
  completion rate 評価は既存 services/phase_gate_evaluator.py (T-008-03) を別 endpoint
  /evaluate-gate で提供済みのため、本 ticket では F-008 spec API surface に focus。
- **auth model**: bearerAuth は本リポジトリでは未実装 (TODO global)。F-008 EARS の
  401/403 は既存 workspace endpoint と同じ `actor_user_id` / `user_id` 慣行で実装:
  - actor_user_id が空文字 → 401 phases.unauthorized
  - workspace 非 member → 403 phases.forbidden
  - role が owner/admin/workspace_admin/ws_admin 以外で write → 403
- **drift fix scope**: drift-summary T-V3-DRIFT-F-008-01/02/03 を本 ticket で 1 batch
  解決 (3 endpoint 全件 router 新設)。
- **pyright baseline**: pyright --strict は既存 codebase 全体に対し 10 errors
  (workspaces.py) / 3 errors (phase_service.py) を baseline 報告するが、いずれも
  _ConnectionWrapper.row_factory 代入の既知パターン。本 ticket の追加コードは
  baseline を増やしておらず、新規 type error は 0。
- **ruff baseline**: routers/workspaces.py に既存の E402 + E701×3 が pre-existing
  (本 ticket 非関連の `_task_stats` 内 inline if 文)。新規追加コードは 0 warning。
