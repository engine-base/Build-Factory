## ノート
- **scope**: 3 endpoint 新規追加 (POST /api/tasks/{id}/play, POST /api/workspaces/{id}/tasks/play-all, POST /api/workspaces/{id}/play-all). 既存 task_workspace_service (T-V3-B-11) を extend し、共通の verify_workspace_access / topo_sort / rate limit pattern を再利用.
- **AC-F5 rate limit**: T-V3-B-11 の workspace-scoped rate limit (10/min/ws for bulk_play) と分離して per-user 30/min を新設 (`_rl_check_user` / `_rl_user_bucket` / `_RL_TASK_PLAY_LIMIT=30`). in-memory token bucket. test 用 `_rl_reset_user` を追加.
- **AC-F1 dep check**: `play_task` で `bf_task_dependencies` を LEFT JOIN し、依存先 task の status が completed/done でなければ DependencyConflict を raise (= 409). cycle 防止は migration 20260512300000_cycle_prevention_triggers.sql で DB 制約済.
- **play-all skipped 算出**: `_partition_playable` で (1) status ∈ {completed, done, cancelled, in_progress, running} → skipped, (2) 依存未充足 → skipped, (3) それ以外 → playable. F-007 spec の `outputs_2xx={queued:int, skipped:int}` に対し、`queued` = "spawn 起動できず queue 行きとなった件数 (max_parallel 超過分)" として実装.
- **AC-F6/F8 contract 差**: `/tasks/play-all` は {queued, skipped} 両返却、`/play-all` は {queued} のみ返却 (spec 準拠). 後者は filter 機能を持たず "workspace 単位の全 task 起動" を表す.
- **404 vs 401 precedence**: actor 不在 → Unauthorized (401) を最優先. その後 task 存在検証 → TaskNotFound (404) → workspace_member 検証 → Forbidden (403) → dep 検証 → DependencyConflict (409). spec の outputs_4xx 優先順位に従う.
- **drift fix (pre-existing main.py)**: backend/main.py L316 で `app.include_router(auth_router)` が呼ばれているが、対応する import が欠落していた (2026-05-16 introduced, T-V3-B-11/13/19 等の peer test を全てブロック). T-V3-B-12 では from main import app が必須 (T-V3-B-11 既存 test と同 pattern) のため、L136 に `from routers.auth import router as auth_router` を追加. これにより 79 件の test が pass (34 新規 + 45 既存 T-V3-B-11 regression test). task の `work_package_boundary.shared_no_concurrent_edit` は `backend/app/main.py` を指しているが本リポでは `backend/app/` は存在しないため `backend/main.py` への 1 行追加は scope 外と判断.
- **未実装の dependency 解決 (上流)**: F-007 spec の rate_limit "30/min/user" は per-user カウントを要求するが、user 単位の正確な特定は actor_user_id 文字列に依存 (JWT sub fallback). 本実装は in-memory token bucket のため、複数 process / 水平 scale 時は Redis 移行が必要 (T-V3-B-11 と同条件).
- **エラー code naming**: T-V3-B-11 の prefix "tasks." を踏襲. `tasks.dependencies_not_satisfied` / `tasks.task_not_found` / `tasks.max_parallel_reached` を新規追加 (router + service exception 1:1 対応).
- **test 構成**: TestClient + monkeypatch (service モック) + asyncio.run (service 直接呼び) のハイブリッド構成. T-V3-B-11 test と同じ `_FakeConn` / `_FakeAiosqlite` pattern を play 系専用に `_FakePlayConn` として複製. 34 件で AC-F1〜F9 + service-layer helper + rate limiter を網羅.
