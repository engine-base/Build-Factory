## ノート
- **layout discrepancy**: tickets.json の `files_changed` は `backend/app/{routers,services,schemas}/violations.py` を指すが、実 repo layout は `backend/{routers,services,schemas}/` (T-V3-B-17 完了時の判断と同一)。本 task では実 layout に合わせて配置 (boundary 趣旨 violations 名前空間に新規作成 + tests は backend/tests/routers/test_violations.py の new file) を維持。
- **scope vs T-V3-B-17**: T-V3-B-17 (Wave 1) が F-012 を 6 endpoint まとめて実装済 (red_lines CRUD/test + violations approve/reject + GET violations)。T-V3-B-18 (Wave 2) は violations 名前空間に 3 endpoint (list / approve / reject) を domain-bounded 形 (services.violations + schemas.violations + routers.violations 3 ファイル分離) で再構成する task。T-V3-B-17 が backend/services/red_lines.py + backend/routers/{red_lines,violations}.py で実装したものを、T-V3-B-18 では「violations 関連を violations.py に集約」「scheme 層を schemas/violations.py に分離」「service 層を services/violations.py に薄い façade で公開」する形にリファクタ。後段の UI task (T-V3-C-*) が violations 専用 module から import する想定。
- **GET /api/workspaces/{id}/violations の移管**: T-V3-B-17 では同 endpoint を backend/routers/red_lines.py に同居実装したが、T-V3-B-18 で domain boundary を整えるため violations.py に移管。red_lines.py 側は移管マーカーコメントのみ残し、duplicate route 登録 (FastAPI は最初に register したものを優先するが lint 警告対象) を回避。
- **schemas/violations.py**: Pydantic 型のみのモジュール (response DTO + request body + error envelope)。54 stmts / 100% coverage。今後 OpenAPI 自動生成連携 (Phase 1.5) で `model_json_schema()` の入口になる。
- **services/violations.py**: services.red_lines の violation 関数を re-export する 23 行 façade。違反操作の domain boundary を明示し、Wave 2 後段の Postgres backend 置換 (E-032 専用 SQLAlchemy repository) 時に置換箇所を 1 ファイルに集約できる。behavior parity は import alias で保証 (`InvalidViolationInput is _rl.InvalidRedLineInput` 等を test_service_facade_errors_are_aliases で検証)。
- **auth header model**: T-V3-B-17 と完全同一 (Bearer token + X-User-Id + X-User-Role)。`workspace_admin` 必須 / Phase 1 SSO/JWT 統合時の差替ポイントは `_validate_auth`。
- **main.py edit**: `app.include_router(auth_router)` が pre-existing で参照されていたが prior merge で import 行が失われていたため、import を 1 行復元 (本 task の boundary は `shared_no_concurrent_edit: backend/main.py` で許可されている最小限編集)。violations_router の wire は既に T-V3-B-17 で完了済 (L132 import / L314 include_router) なので追加変更なし。
