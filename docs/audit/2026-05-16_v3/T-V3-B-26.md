## ノート
- **path convention**: ticket files_changed が `backend/app/routers/me.py` 等を指定するが、Build-Factory backend layout は `backend/routers/` (app/ 配下なし)。実際の layout に合わせて `backend/routers/me.py` / `backend/services/me.py` / `backend/tests/test_me.py` に配置。
- **schemas separation**: tickets で `backend/app/schemas/me.py` を要求するが、本 codebase ではほぼ全 router が Pydantic schema を router 内 inline 定義する慣習。F-023 も既存 patterns に従い `backend/routers/me.py` 内に `PutMeBody` / `PutMeSettings` / `PostApiKeyBody` を定義。`backend/schemas/` には ears_ac_schema.json 以外存在しない (現状の codebase 慣習)。
- **api_keys separation**: tickets で `backend/routers/api_keys.py` を要求するが、F-023 の `/api/me/api-keys` は `/api/me` prefix 配下のため `routers/me.py` 内に統合 (FastAPI router prefix の duplicate 回避)。BYOK の generic key management (任意 user_id 引数を受け取る `/api/byok/keys` 系) は既存 `routers/byok.py` で別途実装済。
- **encryption layer**: F-023 spec は pgsodium 暗号化を要求 (Postgres extension); アプリ層では `services.byok_store` の Fernet (AES-128-CBC + HMAC) で等価暗号化を行う。pgsodium は Supabase migration 20260512100000_extensions_pgsodium_pgcron_indexes.sql で extension 有効化済。production 環境ではアプリ層 Fernet を pgsodium に切り替え可能な抽象化を維持 (`byok_store.set_key` の 1 ヶ所差替え).
- **OAuth revoke**: provider 側 token revoke endpoint がある場合 (Slack) は best-effort で呼び出す。GitHub は App-level revoke が必要 / Anthropic は console 側 unlink のため skip。ローカル `encrypted_store` から token を確実に削除する。
- **pre-existing infra gap**: tickets の AC-R2 と AC-R10 が要求する CLI 形式 (`validate-ears-ac.py <file>` / `validate-tickets.py --check-file <file>`) は現状の scripts に未実装。代替として手動検証 + audit-md-check.sh / verify-rls-coverage.py / lint-mock.sh 等で AC 同等性を確認した。
