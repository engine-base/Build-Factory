## ノート
- Checkpoint 永続化は SessionRecord.metadata["checkpoints"][checkpoint_id] = {checkpoint_id, created_at, status_at_checkpoint, actor_user_id, sdk_session_id} に格納する設計を採用. 既存の `checkpoints` table は LangGraph 用 (BYTEA blob) で session 単位の checkpoint には不適なため metadata JSON に格納し、E-024 sessions テーブルの metadata::jsonb で永続化される (DbSessionStore.finalize_session は status / resume_choice / crash_reason のみ更新する制約があるが、本 PR 範囲では InMemorySessionStore のテストで動作確認、本番では別 PR で sessions.metadata::jsonb persist を追加する).
- AC-F2 (UNWANTED 409): `_RESUMABLE_DB_STATUSES = {paused, crashed}` で限定し、それ以外は SessionStateConflictError → 409 sessions.not_resumable.
- AC-F3 audit emit: rollback 完了時に `_audit("sessions.rolled_back", ...)` を必ず発火. test では fake_emit で captured を検証.
- workspace_admin role 検証は F-001 auth middleware (T-V3-B-01) が担当する設計のため、本 PR では bearer auth + audit emit までを保証.
- backend/main.py は T-V3-B-15 で wire 済みのため touch 不要だが、上流 merge で `from routers.auth import router as auth_router` の import 行が落ちており TestClient で `NameError: auth_router` が出ていたため最小限の 1 行 (`from routers.auth import router as auth_router`) を復元 (これがないと T-V3-B-15 / T-V3-B-16 含む全 router test が collect 不能).
- Test count: 23 endpoint integration + 7 service unit = 30 (要求 >= 10 を大幅超過).
