- [x] AC-S2: UBIQUITOUS: The system shall render kanban as a feature-grouped accordion (each feature_id = one accordion section containing 4 columns: Todo / In Progress / Review / Done), NOT a flat 6-column board (Hermes-flat layout is forbidden per CLAUDE.md §5.5). → impl: frontend/src/components/kanban/AccordionBoard.tsx (one `<details data-kanban-section data-feature-id>` per feature; inside it `KANBAN_COLUMN_ORDER = ["todo","in_progress","review","done"]` maps to 4 `<section data-kanban-column>` columns).
- [x] AC-S3: UBIQUITOUS: The system shall expand only in-progress feature accordions by default; completed (all Done) and not-started (all Todo) features shall be collapsed by default. → impl: frontend/src/hooks/use-kanban-board.ts (`aggregateKanban` sets `defaultExpanded = inProgressCount > 0 && !allDone && !allTodo`) + frontend/src/components/kanban/AccordionBoard.tsx (`<details open={defaultExpanded}>`).
- [x] AC-S4: UBIQUITOUS: The system shall use Lucide icons exclusively (no emoji) for icon-glyph elements (design-tokens.md §8). → impl: page + AccordionBoard + Column only import from `lucide-react` (Kanban / LayoutDashboard / Search / Filter / Play / AlertTriangle / ShieldOff / ChevronDown / ChevronRight / Layers / Check / GitPullRequest). ## Tier 2: Functional

- [x] AC-F1: EVENT-DRIVEN: When the page mounts for an authenticated workspace member, the system shall call GET /api/workspaces/{id}/tasks?group_by=feature and render tasks grouped by feature_id with accordion-friendly metadata. → impl: frontend/src/api/kanban.ts::getKanbanTasks → kanbanTasksEndpoint always appends `group_by=feature`; frontend/src/hooks/use-kanban-board.ts invokes it once on mount via `React.useEffect`; the response is aggregated by `aggregateKanban` and rendered by `AccordionBoard`.
- [x] AC-F2: UNWANTED: If an unauthenticated visitor navigates to this page, the system shall redirect to /login (S-001) and shall not render any task data. → impl: frontend/src/app/(app)/task/kanban/page.tsx — `useEffect` calls `router.replace("/login")` whenever `error.status === 401`; the early-return shell is `aria-hidden` and contains no task DOM.
- [x] AC-F3: STATE-DRIVEN: While data is being fetched, the system shall render a skeleton accordion with role="status" aria-live="polite". → impl: frontend/src/app/(app)/task/kanban/page.tsx::KanbanSkeleton — top-level `<div role="status" aria-live="polite" data-testid="kanban-skeleton">` rendered while the hook's `loading === true`.
- [x] AC-F4: UNWANTED: If the API returns 403, the system shall display a 403 page (S-046) instead of partial data. → impl: frontend/src/app/(app)/task/kanban/page.tsx::KanbanForbidden — when `error?.status === 403` the page renders the inline 403 (with a deep link to `/forbidden` for S-046) and skips both the AccordionBoard and the error toast; no partial data is rendered.
