## 破壊的変更 / 警告

- ⚠ breaking_change: ALTER TABLE rename on production
  - mitigation: migration は `IF legacy_exists AND NOT canonical_exists` guard により本 repository (canonical のみ存在) では no-op. v1 schema を import 済みの operator のみ legacy → canonical rename が発火する. DROP / TRUNCATE / DELETE 一切なし (test_migration_is_non_destructive で静的検証).
  - rollback: 各 rename は `ALTER TABLE canonical RENAME TO legacy` で逆方向に手動巻き戻し可能. BEGIN/COMMIT transaction で囲んでいるため migration 中の失敗は自動 rollback.
