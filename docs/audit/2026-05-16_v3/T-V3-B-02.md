## ノート
- **Project layout deviation (承継)**: ticket spec の files_changed は `backend/app/routers/...` 記法だが、実 project layout は `backend/{routers,services,schemas,tests}/` (T-V3-B-01 commit a88e4d8 のメッセージにも同記載あり). 本タスクも同 layout に従う.
- **main.py wiring fix**: 着手前確認で `app.include_router(auth_router)` (line 316) は参照しているが import が main.py から欠落しており `NameError: name 'auth_router' is not defined` で T-V3-B-01 含む既存 auth tests が import 不能だった. work_package_boundary.shared_no_concurrent_edit に `backend/app/main.py` がリストされているが minimal な 1-line import 追加で対応 (`from routers.auth import router as auth_router`).
- **MFA endpoints status code (201 vs 200)**: openapi.yaml では `mfa/enroll` と `mfa/verify` ともに `201 Created` を採用. router の status_code に合わせる (login は 200, signup は 201, password-reset は 200, mfa/* は 201, oauth/callback は 200).
- **MFA verify auth role**: features.json#F-001 で `mfa/verify.auth = "public"` (mfa_verify は MFA challenge flow の続きで access_token を発行するため). AC-F7 (UNWANTED 401) は "malformed Authorization header forwarded → 401" を意味すると解釈 (T-V3-B-01 login AC-F5 と同パターン). 完全に missing なヘッダは 401 ではなく業務ロジックへ進む.
- **OAuth callback auth role**: features.json#F-001 で `oauth/callback.auth = "public"`. AC-F11 (UNWANTED 401) も同パターン.
- **TOTP impl**: RFC 6238 standard (SHA1 / 30s step / 6 digits / ±1 step window). Base32 secret は RFC 4648 alphabet (A-Z2-7) で 16-128 桁を許容. Backup codes は 8 hex chars × 8 個.
- **OAuth state CSRF**: in-memory OAuthStateStore で issue/consume を実装. state は single-shot (replay 防止). TTL 10 min. real provider exchange は services/oauth_providers.py (T-023-04) に委譲する想定だが、本 task では F-001 contract の access_token + refresh_token + user_id 発行までを scope に. user_id は (provider, code) hash から deterministic UUID を生成.
- **Test count**: 28 tests added (Gate 5 要件 ≥ 10 を大幅に超過). 全 12 AC を網羅 + supplementary regression (provider enum 4 通り walk-through, TOTP window tolerance, route discovery).
- **Schema validation precedence**: FastAPI は body schema validation (Pydantic) を dependency resolution の前に実行するため AC-F4 (auth before validation) と AC-F5 (validation 422) の順序は厳密には FastAPI 実装依存. AC-F4 は dependency 抽出後の最初の処理として明示再チェック.
- **MFA validate-ears-ac script**: validate-ears-ac.py の TICKETS_PATH が hardcoded で 2026-05-09_v1 を指しているため T-V3-B-02 (2026-05-16_v3_phase1) を直接走らせると task_id not found になる. これは pre-existing script limitation で Gate 2 の代替として scripts/validate-tickets.py を使用.
