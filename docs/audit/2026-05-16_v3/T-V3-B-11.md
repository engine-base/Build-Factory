## ノート
- impl 配置: openapi `x-bf-implementation-path` は `backend/routers/workspaces.py::...` を指しており、tickets.json files_changed の `backend/app/routers/tasks.py` (modify) は誤り (実 repo に `backend/app/` 層なし). 実装は workspaces.py に追記し、新 service `backend/services/task_workspace_service.py` を新規作成.
- entity ID: audit テンプレ初版の "E-016 Task / E-017 Dependency" は entities.json では "E-014 Task / E-015 TaskDependency" が正. RLS coverage は entity ID に関わらず table 名 (bf_tasks / bf_task_dependencies) に対し PASS.
- rate limit: in-memory token bucket (10/60s/workspace). 単 process 用. 横スケール時は redis-based に置換予定 (ADR で別途記録).
- auth: 既存 patterns に合わせて `actor_user_id` body/query 経由. JWT/bearer 検証は scope 外 (T-V3-B-XX で別途 / ADR-014 予定). actor 不在 → 401, non-member → 403, workspace 不在 → 404 を service 層 exception で一元 map.
- topo sort: cycle 検出は DB trigger (T-001-09) が担当. service は residue を末尾に積むだけ (DAG エディタが上流で防止前提).
- ws → task scope: `bf_tasks` は project_id 経由で workspace に紐付くため `JOIN bf_projects p ON p.id=t.project_id WHERE p.workspace_id=?` でフィルタ. 直接 workspace_id 列ではない.
