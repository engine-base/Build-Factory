## ノート
- entity_ids "E-035 KnowledgeItem / E-036 ObsidianVault" は ticket 上の論理 ID. 実際の table は v3 entities.json で E-068 KnowledgeBase (`knowledge_base`) / E-033 ObsidianVault (`obsidian_vaults`).
- ticket 指定パス `backend/app/routers/knowledge.py` は当 repo の実構造 `backend/routers/knowledge.py` に置換 (backend/app/ ディレクトリは存在しない). 同様に `backend/app/services/knowledge.py` → `backend/services/knowledge.py`, `backend/app/schemas/knowledge.py` → router 内 Pydantic model に統合 (既存 routers の慣例).
- 認証は既存 routers の慣例 (Authorization Bearer header OR `?user_id=` query) を踏襲. JWT 検証本体は middleware / RLS layer に委譲. AC-F3 / AC-F6 401 はヘッダ・パラメタ両方欠落で raise.
- DB 不在 (test env) では service レイヤで graceful に空 list を返し、router は 200 + 空 contract を維持. これにより 401/422 と 2xx を separation of concerns で test 可能.
- pgvector embedding は services.embedding_service.embed の API key 未設定時 None fallback を許容. None 時は SQL を pg_trgm + ts_rank の合成のみに切替.
- main.py は `routers.knowledge` を `app.include_router(knowledge_router)` で登録 (knowledge_actions_router の直後).
