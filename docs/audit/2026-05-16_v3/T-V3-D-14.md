## 破壊的変更 / 警告

- ⚠ schema_change: auth_audit_log table → backward-compat VIEW (security_barrier=true)
  - 対応: 旧 table は `_archived_auth_audit_log` に RENAME し物理 DROP しない (history 保全). 1 release cycle (Phase 2 初) で FOLLOWUP-1/2 task で 410 Gone deprecate → 物理削除予定. ADR-018 §Decision 2 / §Consequences 参照.
  - migration idempotent guard: `pg_class.relkind='r'` チェックで「既に view 化済」「table 未存在 (fresh DB)」を NOTICE で skip.
- ⚠ access_change: auth_audit_log の旧 owner_self_select policy → audit_logs 側 RLS 継承
  - 対応: 旧 `auth_audit_log.audit_self_read` (本人のみ SELECT) は VIEW 経由では audit_logs の RLS (`audit_logs_account_member_select`) に従う. `workspace_id IS NULL` の auth row は service_role 経由のみ可視 (admin/SOC は専用 API で). ADR-018 §Decision 3 で意図的判断として明記.
- ⚠ insert_path_change: 新規 auth event は `audit_logs(source='auth')` に書く
  - 対応: backend/services/audit_service.py::emit_auth_event が backward-compat wrapper. 旧 `auth_audit_log` への直接 INSERT する production コードは存在しなかった (grep auth_audit_log で hits は test/migration/comment のみ確認). ADR-018 §Decision 4.
