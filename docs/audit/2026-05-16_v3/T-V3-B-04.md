- **認可**: Phase 1 では Authorization Bearer / SupabaseAuth 統合前のため `actor_user_id` 必須化 = 401 で表現 (`_require_actor`). T-V3-B-04 audit MD AC-F5/F7/F11 の解釈と整合.

- **clone opt-in**: supabase ai_clones table の is_opted_in / opted_in_at と整合する registry. opt_in_acknowledged=False or user 未承諾は両方 403 (CloneOptInError → AC-F2).
- **org-chart**: parent_employee_id (E-007 schema 整合) を AIEmployee dataclass に追加. delete 時は FK SET NULL (cascade) 相当の挙動. include_inactive デフォルト False (= 非 archived only / AC-F1).
- **audit emit**: `ai_employee_invocation` (test) / `ai_employee_clone` (clone-from-user) を F-003 audit_logs spec と整合.
- **entity ID 揺れ**: ticket spec は E-008/E-021/E-022 を AIEmployee/Skill/SkillExecution として記載しているが、entities.json v3 では E-007=AIEmployee, E-008=Skill, E-021=ArtifactVersion, E-022=Screen, E-044=AIClone, E-046=AIPersona. 実装は entities.json v3 が正 (supabase migration が source of truth).
