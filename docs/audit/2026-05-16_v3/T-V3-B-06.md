### 実装方針
- **PUT /api/workspaces/{id}/members/{user_id}/role**: 既存 PATCH /api/workspaces/{id}/members/{user_id} (custom_permissions 含む汎用更新, T-021-05) と並列で、OpenAPI 仕様 F-004 (T-V3-DRIFT-F-004-06) に準拠した role 専用ミニマル契約を提供. backend/services/workspace_service.update_member_role は T-021-05 で既に self-strip / owner-protection を持っており再利用.
- **DELETE /api/workspaces/{id}/invitations/{token}**: workspace_service に新規 `revoke_invitation` を追加. status='pending' のみ revoke 可能, accepted/expired/already-revoked は `InvitationRevokedError` (409). cross-workspace token は `InvitationNotFoundError` (404). schema 変更なし (status='revoked' を使う; revoked_at は service-layer で生成).

- **auth 401**: require_user dependency を両 endpoint に貼ることで `BUILD_FACTORY_DEV_BYPASS_AUTH=0` 時 401 を保証 (AC-F8 / AC-F11).
