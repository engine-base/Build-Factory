## ノート
- 実装パス調整: tickets.json は `backend/app/routers/onboarding.py` 表記だが、 actual codebase 構造は `backend/app/` を使わず `backend/{routers,services}/` 直下なので convention に合わせて `backend/routers/onboarding.py` / `backend/services/onboarding.py` / tests は `backend/tests/test_onboarding.py` (既存 lifecycle pattern) に配置.
- Pydantic schemas は別ファイル `backend/app/schemas/onboarding.py` を作らず router 内に inline (既存 routers の慣習に従う: account_settings.py / bf_profile.py 参照).
- DB: psycopg/Postgres アダプタ (`db.async_db`) 経由 + `user_onboarding_state` 新規 table (alembic migration `backend/alembic/versions/i6c7d8e9f0a2_onboarding_state.py`). 既存 user_settings table はスキーマ readonly のため、 onboarding state は専用 table 化.
- 3 step 構造: welcome (required) → workspace_setup (required) → ai_employee_intro (optional). required step を skip しようとすると 409 (AC-F2).
- audit event: `onboarding.advanced` / `onboarding.completed` / `onboarding.skipped` を `services.memory_service.emit_event` 経由で audit_logs に silent emit (failure 時も本処理は止めない).
- 401 検証 test は `DEV_BYPASS=False` を monkeypatch で強制 (env var では import-time に capture 済).
