- SessionStore は agent_runner singleton 経由で取得し、本番では DbSessionStore / テストでは InMemorySessionStore に monkeypatch する pattern を踏襲 (T-010b-05 と同一 fixture 構造).
- `_list_all_records` ヘルパーは InMemorySessionStore (`_sessions` dict) と DbSessionStore (`list_sessions()` 想定) を両対応するアダプタ層。後者の実装は T-V3-B-16 もしくは別 backend gap task の範疇とする.

- `_require_auth` は Bearer 形式の最小境界条件のみ強制 (token decode / JWT 検証は F-001 / T-V3-B-01 が担う).
