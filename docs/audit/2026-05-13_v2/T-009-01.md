### 2.3 Therefore T-009-01 task_dependencies CRUD =

T-009-01 (BE / REUSE / 0.3d) は F-009 の「DAG edges 永続化層」を担当する CRUD API. 具体的に以下のサブ要件を満たす:

1. **エッジ作成 (POST)**: タスク追加時に「依存選択 → DAG 可視化」のために `task_id → depends_on_task_id` のエッジを INSERT (happy_path).
2. **エッジ取得 (GET by task / GET by project)**: DAG 可視化 + 影響範囲ハイライトのため、task 単位 / project 単位で読み取れる. S-017, S-029 のフロント描画のソース.
3. **エッジ削除 (DELETE)**: PM が「skip」判断時にエッジを除去できる.
4. **dep_type enum**: `blocks` / `related` / `informs` — happy_path の「依存選択」UI が選ぶ種別.
5. **循環依存 → DB 制約 + 4xx**: error_paths の「循環依存→trigger で block」を T-001-09 trigger (`bf_prevent_task_dep_cycle`) に委譲し、サービス層は `DepCycleDetected` で 409 に変換.
6. **自己参照禁止**: CHECK `no_self_dep` (DDL) + サービス層先回り validation.
7. **DUPLICATE 禁止**: UNIQUE `uq_bf_dep (task_id, depends_on_task_id)` → 409 `dep_duplicate`.
8. **RLS / workspace 境界**: T-001-04 + T-001-06 で member 単位 RLS を継承.
9. **エラー封筒 `{detail: {code, message}}`**: 全 4xx 共通形式 (T-001-02 で共通定義).

なお「大規模 DAG → 階層折りたたみ」「AI 影響範囲ハイライト」は UI 層 (S-017 / S-029) と AI 層 (別タスク) の責務であり, T-009-01 (BE CRUD) のスコープ外.
