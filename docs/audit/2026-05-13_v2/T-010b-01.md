## post-mortem / lessons applied from PR #253

PR #253 で学んだ「strategy ラベルが偽装される / Phase boundary が曖昧」反省を
本 audit に直接反映:

1. **precedence layer は 1 件 1 テスト**:
   ADR-012 §5.2 6-layer (header / session / workspace / BYOK / default / fallback)
   を **層ごとに分離した 6 spec test** で固定. 上位層が下位層を必ず override する
   ことを 1:1 で証明.
2. **REFACTOR invariant test を最初に書く**:
   既存 `task_executor / skill_runner / orchestrator_graph` の public signature を
   `inspect.signature` で固定. 後続 PR が signature を破壊したら spec test が
   即 fail する.
3. **drift guard を 4 件配備**:
   ADR-010 禁則 (LangGraph / LangChain / LiteLLM-in-main-runner) を import grep
   で機械検出. sentinel comment (`NO_LANGGRAPH_IN_RUNNER`) も protected.
4. **REFACTOR スコープを honest に明示**:
   本タスクは「既存 task_executor + skill_runner を直接書き換える」のではなく、
   T-S0-08 で配備済の `claude_agent_runner.py` + ADR-012 modules を **既存
   task_executor / skill_runner と並走する追加経路** として audit する scope.
   既存 modules を 1 byte も変更しないことが REFACTOR invariant.
