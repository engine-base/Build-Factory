## Spec literal expansion (源泉 cited from `tickets.json#T-S0-08`)

ticket は 7 AC を持つ. 全文を以下に literal copy し、各文を sub-clause に分解する.

### AC-1 UBIQUITOUS

> "backend/integrations/claude_agent_runner.py shall import claude_agent_sdk (Anthropic official) — explicitly NOT langgraph / langchain — to spawn Claude Code as a subprocess and capture stdout/stderr via the SDK's async message stream; the module shall expose ClaudeAgentRunner, SessionRecord, CostRecord, AuditEvent, SessionStore (+ DbSessionStore / InMemorySessionStore), CostHook (+ DbCostHook / NoopCostHook), SummaryHook (+ DbSummaryHook / NoopSummaryHook), AuditHook (+ DbAuditHook / NoopAuditHook), and VALID_RESUME_CHOICES."

→ 17 public symbol + claude_agent_sdk import 文字列の存在 + Db/Noop の subclass 関係.

### AC-2 UBIQUITOUS

> "ClaudeAgentRunner.run() shall accept an sdk_session_id parameter (Optional[str], default None) and pass it via the SDK's resume mechanism so the SDK auto-restores conversation history and applies 3-tier compaction; the runner shall NOT re-implement the SDK's auto-compaction (ADR-010 §自前実装禁止)."

→ run_task の sdk_session_id 引数 / SDK options.resume への伝搬 / SDK auto-compaction (9 section summary / tool trim) の **不再実装**.

### AC-3 EVENT-DRIVEN

> "When ClaudeAgentRunner.run_task() is invoked, the system shall create a SessionRecord with status='running' via SessionStore.create_session() within 2 seconds, then stream each message line to session_logs via SessionStore.append_log(); the SessionRecord.status field shall be constrained to one of (running, done, crashed, cancelled, paused) consistent with supabase/migrations/20260510000003_runner_session_tables.sql CHECK constraint."

→ create_session 呼び出し / status default = running / append_log per message / 5-status enum (migration と一致).

### AC-4 STATE-DRIVEN

> "While the runner is active, ClaudeAgentRunner.cache_ttl_seconds shall default to 300 (5min ephemeral) so the SDK applies prompt cache automatically; the system shall record cache hit rate / input_tokens / output_tokens to cost_logs via CostHook.record() (DbCostHook implementation INSERTs into cost_logs with cache_read_tokens / cache_write_tokens columns per migration schema)."

→ cache_ttl_seconds default 300 / override 可 / Anthropic Usage 4 値 (input/output/cache_read/cache_write) / DbCostHook SQL に migration column 全列.

### AC-5 STATE-DRIVEN

> "While context usage exceeds 95%, the SDK shall auto-generate a 9-section structured summary which the system shall persist via SummaryHook.persist() to chat_messages.compressed_summary (JSONB column per migration); the runner shall NOT generate the summary itself (SDK auto-feature reuse, ADR-010)."

→ SummaryHook abstract が NotImpl raise / DbSummaryHook が chat_messages.compressed_summary に書き込み (migration が JSONB 列) / runner module に build_summary / compress_context / SECTION_KEYS なし.

### AC-6 EVENT-DRIVEN

> "When the subprocess crashes (exception thrown by the SDK message loop), the system shall set SessionRecord.status='crashed' with crash_reason set, and ClaudeAgentRunner.handle_resume(choice) shall validate choice against VALID_RESUME_CHOICES = ('from_checkpoint', 'rerun_full', 'manual_fix', 'cancel') raising ValueError on invalid input; supabase migration sessions.resume_choice CHECK constraint shall mirror the same 4 values."

→ **(CRITICAL anti-drift)** 4 choice の固有挙動を 1:1 で個別検証:

| choice | runner 挙動 | 終端 status | run_task 呼ぶか | sdk_session_id |
|---|---|---|---|---|
| `cancel` | finalize のみ | `cancelled` | NO | (未変更) |
| `manual_fix` | finalize のみ | `paused` | NO | (未変更) |
| `from_checkpoint` | run_task に **元 sdk_session_id 渡し** | `done` (新 record) | YES | resume=元 id |
| `rerun_full` | run_task に **sdk_session_id=None 渡し** | `done` (新 record) | YES | resume=None |

### AC-7 UNWANTED

> "If `from langgraph` / `import langgraph` / `from langchain` / `import langchain` appears in backend/integrations/claude_agent_runner.py or any module on the main runner path, scripts/lint-mock.sh --no-langgraph shall exit non-zero (ADR-010 forbids them in main path); the pytest module test_t_s0_08_runner_invariants.py shall mechanically assert the import is absent."

## post-mortem (Wave 5 を通じて見えた pattern)

**T-S0-08 は T-013-04 / T-010b-01 と比べて "production はすでに正しい / test layer だけ追補" 型タスク** だった点が特殊:

- 既存 production code (runner) は ADR-010 の意図通り 1 modular file に整理されており、 spec drift は無かった
- 4-choice resume の test が parametrize で collapsed されていた潜在的 drift だけが残っていた → Wave 5 で 1 choice = 1 test で fixed
- **prompt 強化 takeaway** (T-013-04 post-mortem を継承):
  - 「strategy / variant / mode 系の引数 (今回は `choice`) を取る関数では、各 variant が実際に異なる挙動を発生させる test を **1:1 で書け**」 — このルールが Wave 5 spec layer の核
  - 「3 component 構造 (今回は 3-tier compaction の trim / cache / summary) は責務分担を test 側でも 3 sub-clause に分解せよ」
