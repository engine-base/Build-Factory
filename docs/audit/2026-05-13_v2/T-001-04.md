## post-mortem

**新 pre-flight audit workflow の有効性**:
- collapsed parametrize regex を解いて 11 個別 test に分けることで、どの 1 テーブルが drift しても test name から直接特定できる。
- RLS policy 名 1:1 enumeration (双方向 drift guard) により、policy rename / drop / 追加が audit doc 更新を強制する設計に。
- audit_logs.workspace_id **nullable invariant** のような「うっかり強化してしまう変更」を逆方向 drift guard で検出する設計は有効。

**今後の prompt 強化点**:
- DDL audit では「列挙が 5 以上の集合は parametrize ではなく個別 test として展開せよ」を明文化。
- RLS / policy / trigger / function 名は audit doc に必ず full list をピン留めし、双方向検証 test を 1 件ずつ書く。
