プロジェクト概要

作るもの

Build-Factory（開発工場 OS）

ヒアリング → 要件定義 → 設計 → タスク分解 → 実装 → テスト → 進捗管理 → 納品 までを 1 つの Web アプリで完璧に進める。

本質的な動機

1 人で 10 案件並走しても破綻しない開発体制を構築し、SaaS 商用化する。

⚠ コンセプトの再定義（v2）

v1 は「決めるのは人間 / AI が実行」をコンセプトにしていたが、これは設計思想に過ぎない。 コンセプト = 「企画から納品までの全工程を 1 環境で完璧に進めるプロジェクト管理 OS」

成功の定義

定量

  • 自社 1 案件を End-to-End 完走
  • 1 人で 10 案件並走を破綻なく
  • AI 出力差し戻し率 80% → 20% 以下
  • 1 タスク Token コストが workspace 上限内
  • SaaS マルチテナント・認証・権限が商用提供レベル

定性

  • 「もう手放せない」状態
  • 情報過多にならない（progressive disclosure）
  • 監視担当が「確認だけで OK」と言える
  • 短期に早く / 長期で負債を残さない

アーキテクチャ全体像（7 レイヤー）

L1 UI

Next.js 15 + shadcn/ui + Recharts + React Flow / HTML+MD ハイブリッド / GrapesJS embed / progressive disclosure

L2 AI 社員（ハイブリッド）

BMAD ペルソナ + Anthropic Agent Teams + 既存 96 スキル

L3 プロジェクト管理（独自 moat）

フェーズ管理 / 依存グラフ / 影響分析 / 自動伝搬 / 多 view（Kanban/List/Gantt/DAG/Swarm）

L4 実装連携

Claude Code（OAuth で各自プラン）+ MCP / Anthropic Agent Teams Plan-Gen-Eval / ▶︎ 再生 + 並列 swarm + Hermes 風 circuit breaker

L5 連携

Slack 双方向 + Claude Desktop MCP / GitHub / Obsidian Headless Sync / デプロイ連携アダプタ

L6 データ + 認証 + 観測

Supabase（Postgres + Auth + RLS + Realtime + Storage）/ Langfuse self-host / OpenFGA（Phase 2）

L7 ナレッジ + LLM

Obsidian 母艦 + Mem0 + 自動キュレーション / 実装 = 各自 Claude Pro/Max / チャット = 自社 API or BYOK
