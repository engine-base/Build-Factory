インデックス方針
必須インデックスパターン（全テーブル共通）
パターン例理由
RLS 対象カラムidx_X_workspace_id ON X(workspace_id) WHERE deleted_at IS NULLRLS 性能
workspace_members 評価idx_wsm_user_workspace ON workspace_members(user_id, workspace_id) WHERE deleted_at IS NULLRLS で頻繁参照
全 FKidx_tasks_phase_id ON tasks(phase_id)JOIN 性能
特殊インデックス（用途別）
用途採用適用テーブル
全文検索（M-24）GIN on tsvectortasks / artifacts
semantic 検索（M-24）ivfflat (vector_cosine_ops)tasks / artifacts / user_interaction_log
fuzzy 検索GIN (gin_trgm_ops)tasks.title / artifacts.title
配列検索GINai_employees.skill_ids / tasks.tags
時系列（追記専用）BRINaudit_logs / session_logs / cost_logs
partial（部分条件）WHERE is_current = trueconstitutions.is_current
partial（unread）WHERE read_at IS NULLnotifications
複合インデックス（クエリパターン特化）
クエリインデックス
カンバン viewtasks(workspace_id, status, priority DESC, created_at DESC) WHERE deleted_at IS NULL
swarm 実行中一覧sessions(workspace_id, status, started_at DESC) WHERE status IN ('starting','running','paused')
依存グラフ traversaltask_dependencies(parent_task_id, child_task_id) + (child_task_id, parent_task_id)

設計ノート
項目方針
PK 型UUID v7（時系列ソート可・衝突なし）
削除戦略ソフトデリート（deleted_at TIMESTAMPTZ）+ RLS で auto フィルタ / 物理削除は admin 手動のみ
マルチテナントaccount_id / workspace_id 全テーブル必須 + RLS で auto enforce
命名規則snake_case / 複数形 / FK は <table>_id
タイムスタンプcreated_at / updated_at（trigger 自動更新）
作成・更新者追跡created_by / updated_by UUID FK→users
JSONB 活用custom_permissions / project_meta / capabilities / criteria / before / after
enumPostgres ENUM 型 + check constraint で型安全
監査連携主要テーブルに変更検出 trigger → audit_logs に自動 INSERT
時系列テーブルaudit_logs / session_logs / cost_logs は BRIN + pg_partman 月次 partition（Phase 2）
暗号化API キー / OAuth トークンは pgsodium で bytea 列に保管 / UI は masked 表示
マイグレーションSupabase CLI（supabase migration new + supabase db push） / 既存 alembic は archive
seedsupabase/seed.sql / BF_ENV=development ガード必須
本番マイグレ戦略expand-and-contract（zero-downtime）
個人クローン opt-inusers.user_clone_opt_in DEFAULT FALSE + INSERT trigger で強制
