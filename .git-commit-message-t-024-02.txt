## Type C findings (3 silent-fail bugs — fix in follow-up T-024-02b)

Manual repro confirms 3/4 source handlers always return [] due to defensive
`except Exception` swallowing:

1. `_search_knowledge` calls `embedding_service.search_knowledge(q=..., account_id=...,
   workspace_id=..., limit=...)` but real signature is `(query, skill_tags=None,
   top_k=15, min_score=0.4)` → TypeError every call.
2. `_search_tasks` imports `services.bf_db` but no such module exists →
   ImportError every call.
3. `_search_screens` has same bf_db missing-import problem.

Existing test fixture stubs all handlers so spec tests PASS but real I/O fails
silently. AC-3 isolation is met formally; dataloss is total. Recommend
follow-up task T-024-02b (cf. T-013-04 → T-013-04b pattern in PR #253).
